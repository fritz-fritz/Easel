# Security policy

Easel is pre-alpha. Security fixes are currently provided only for the latest revision of the
`main` branch.

Please do not open a public issue for a suspected vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/fritz-fritz/easel/security/advisories/new)
and include reproduction steps, affected platforms, and any known mitigations.

Do not include real credentials, personal images, private filesystem paths, or signing material
in a report. We will acknowledge a complete report as soon as practical and coordinate disclosure
after a fix is available.
